## Problem
Video files were creating 3x memory copies through Base64 encoding/decoding.
- 100MB video consumed ~467MB memory

## Solution
Store video bytes directly instead of Base64 encoding.
- 100MB video now consumes ~100MB memory (79% reduction)

## Changes
- `Video.java`: Added `byte[] data` field and `fromBytes()` method
- `OpenAIVideoModel.java`: Line 108 - uses `Video.fromBytes()`
- `GeminiVideoModel.java`: Lines 160-167 - uses `Video.fromBytes()`
- `OpenAI.java`: Lines 244-257 - uses `getData()` with fallback
- `GeminiVertex.java`: Lines 204-221 - three-tier fallback (data → b64 → URL)
- Tests: Added `VideoMemoryTest` and `VideoProviderMemoryTest` (16 tests, all passing)

## Backward Compatibility
Falls back to base64 decode if `data` field is null.
